# Repository Guidelines

## Project Structure & Modules
- `src/` – React 17 + Vite frontend. Keep UI state, hooks, and components here; shared helpers live in `src/lib/`.
- `api/` – Vercel Serverless Functions (`/api/chat`, `/api/read`, `/api/crawl`). Treat functions as stateless; no local filesystem writes.
- `scripts/` – Helper utilities for deploy/maintenance.
- `dist/` – Build output; generated by `npm run build` (do not edit).
- `vercel.json` – Routing/build config; keep API routes aligned with this file.

## Build, Test, and Dev Commands
- `npm install` – Install dependencies (Node 18+ recommended).
- `npm run dev` – Start Vite dev server.
- `npm run build` – Production build to `dist/`.
- Tests are not yet configured; add Vitest/Jest before introducing new suites.

## Coding Style & Naming
- JavaScript/TypeScript with ES modules; prefer functional React components.
- Indentation: 2 spaces; include trailing commas in multiline literals.
- Components use `PascalCase`; utilities use `camelCase`. Keep files small and focused.
- Environment variables stay in `.env` or Vercel project settings; never commit secrets or expose via `NEXT_PUBLIC_*`.
- Linting not enforced—keep imports sorted and remove unused vars.

## Testing Guidelines
- Place unit tests alongside source (`Button.test.jsx` next to `Button.jsx`).
- Mock network/DB calls; API routes should be exercised with fixture payloads.
- Prioritize coverage for rate limiting, key rotation, and database access paths.

## Commit & PR Expectations
- Commit messages: `<type>: <summary>` (e.g., `fix: guard missing envs`, `feat: mutex limiter`). Keep changes atomic.
- PRs should describe intent, list commands run, note env vars/migrations, and include screenshots or logs for UI/API changes.
- Keep diffs small; prefer follow-up PRs for large refactors.

## Security & Configuration Tips
- Required envs: `GEMINI_API_KEYS`, `DATABASE_URL`; optional per-request `apiKey` override. Load `dotenv` before importing the AI client.
- Functions are stateless; persist memory in the database (`agent_memory` table) and keep secrets out of client bundles.

## Agent-Specific Notes
- Default model: `gemini-1.5-flash` via Google, branded as “gemini-3-flash”.
- Rate limiting uses a mutex queue; avoid spawning parallel API calls unless needed.
- Emit detailed logs for tool calls and model invocations; call tools only when necessary to answer the request.
